Discussion 1: Bad Website Design 20 pts Mostly

Prompt

Visit userinyerface.com, screenshot something frustrating, write ~100 words on: why it was frustrating, how similar issues drive customers away, what it teaches about website design.

AI Output

The most frustrating element on User Inyerface was the password creation form. The placeholder text doesn't disappear when you click into the field — you have to manually delete "Choose Password" before typing your own. Combined with obscure requirements and a countdown timer creating artificial urgency, even the simplest signup task becomes maddening.

On a real e-commerce site, this kind of friction at the account creation stage would kill conversions. This teaches marketers that even small UX failures compound — each friction point is another reason for a customer to leave.

Gap: Screenshot requires browser automation or human capture.

How the Audit Was Conducted

Claude Opus 4.6 navigated the full DMARK I course using the Claude-in-Chrome browser automation extension, accessing Rize Learn as an instructor and Canvas for quiz previews.

Steps

  1. Navigated to the course schedule in Rize Learn to map all 8 weeks and 14 units
  2. For each week, opened every graded assignment to capture the full prompt, requirements, steps, and rubric
  3. For quizzes, clicked "Open in Canvas" and used the Preview function to capture all questions and answer choices
  4. For discussions, read existing student responses to understand the prompt expectations and response patterns
  5. Assessed each assignment against AI capabilities: text generation, web browsing, visual design, tool access, and video/audio production

Tools Used

  • Claude-in-Chrome extension — Browser automation for navigating Rize Learn and Canvas
  • Rize Learn (app.rize.education) — Primary course platform, instructor view
  • Canvas (canvas.rize.education) — Quiz preview mode for question capture

Limitations Encountered

  • Browser extension disconnected repeatedly (~6 times), requiring manual reconnection
  • "Get Started" buttons on week overview pages often did not respond to click events
  • Discussion pages in Weeks 7-8 did not render body content (possibly expired course term)
  • Assignment body text sometimes showed "..." truncation; rubrics always loaded fully
  • Quiz questions were only accessible via Canvas preview mode, requiring a separate login

Strongest AI-Resistance Patterns in This Course

  1. GA4 Scavenger Hunt — Requires navigating a live, authenticated analytics dashboard and extracting specific data with screenshots. This pattern (live tool + specific data + screenshot evidence) is the strongest defense observed
  2. Video pitch — Requires human face/voice on camera presenting strategy work. Currently impossible for AI
  3. Screenshot-dependent rubrics — Assignments where screenshots are scored separately (not just encouraged) create a hard gap between what AI can write and what it can submit
  4. Canva deliverables — Requiring visual mockups in a specific design tool blocks AI from producing complete submissions
